# Smash Hit

Небольшой Unity-прототип, вдохновлённый игрой Smash Hit. Игрок бросает металлические шары в летящие навстречу стеклянные панели, разбивает их и старается не допустить, чтобы стекло перекрыло путь камере.

Сейчас проект находится на стадии игрового прототипа с уже собранным базовым core loop: бросок, попадание, разрушение стекла и проигрыш при блокировке прохода.

Работу выполнил Голубев Тимофей, группа М8О-406Б-22

## Что сделано

- Собрана игровая сцена `Gameplay` с атмосферным окружением, светом и постобработкой.
- Реализован бросок шаров мышью и через touch-ввод.
- Добавлен спавн стеклянных панелей с разными размерами и положением.
- Сделано процедурное разрушение стекла на осколки при попадании.
- Поддержаны как отлетающие осколки, так и оставшиеся висеть части стекла.
- Добавлено базовое состояние проигрыша, если стекло или осколки перекрывают путь камере.

## Как работает разрушение стекла

Разрушение сделано процедурно. В момент удара для стеклянной панели вычисляется точка попадания, после чего внутри формы стекла генерируется набор точек вокруг центра удара. По этим точкам строятся фрагменты, из которых получается карта будущих осколков.

Дальше алгоритм работает так:

- область стекла делится на полигоны вокруг точки попадания;
- для осколков строится граф соседства, чтобы понять, какие части всё ещё связаны с краями стекла;
- осколки рядом с точкой удара принудительно отсоединяются, даже если формально ещё имеют связь;
- большие связанные фрагменты остаются как "подвешенные" части стекла;
- отделившиеся осколки получают импульс от удара, гравитацию и физическое движение.

Если игрок попадает по оставшемуся поддерживаемому куску, он может расколоться ещё раз и породить новые более мелкие осколки.
